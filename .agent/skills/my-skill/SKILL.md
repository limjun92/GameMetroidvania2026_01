# 코드 리뷰 스킬

코드를 리뷰할 때 다음 단계를 따르세요:

## 리뷰 체크리스트

1. **정확성**: 코드가 의도한 대로 동작합니까?
2. **엣지 케이스**: 오류 상황이 처리되었습니까?
3. **스타일**: 프로젝트 규칙을 따릅니까?
4. **성능**: 명백한 비효율성은 없습니까?

## 기록 관리

- 작업 전 `History` 폴더가 없다면 생성합니다.
- `History/YYYY_MM_DD.md` 형식(예: 2024_01_01.md)으로 오늘 날짜의 파일을 생성하거나 엽니다.
- 코드 변경 사항, 수정 이유, 주요 로직 설명 등을 해당 파일에 업데이트합니다.

## 필수 코딩 체크리스트

코딩을 수행할 때 다음 사항을 반드시 확인하고 준수하세요:
1. **에러 처리**: 예기치 않은 오류나 예외 상황에 대한 처리가 누락되지 않았는지 확인.
2. **로그 남기기**: 디버깅과 추적을 위해 적절한 위치에 로그가 있는지 확인.
3. **변수/함수 네이밍**: 의도가 명확하게 드러나는 이름을 사용했는지 확인.
4. **하드코딩 지양**: 설정값이나 상수는 별도로 분리되어 있는지 확인.
5. **불필요한 코드 제거**: 사용되지 않는 변수나 주석 처리된 코드가 없는지 확인.

## 피드백 제공 방법

- 변경이 필요한 부분에 대해 구체적으로 작성하세요
- '무엇'뿐만 아니라 '왜' 변경해야 하는지 설명하세요
- 가능한 경우 대안을 제시하세요

## 언어 규칙

- 항상 한국어로 답변하세요
- implementation plan을 작성할 때는 항상 한국어로 작성하세요

## 주석 작성 규칙 (2026-02-07 추가)
- 스크립트를 수정하거나 생성할 때, 해당 **스크립트의 역할** class 위에 주석으로 설명하세요.
- 주요 **함수의 역할**과 복잡한 로직에 대해 주석을 꼼꼼하게 작성하세요.
